Backend

Laravel 12 (PHP 8.2/8.3), monólito com Inertia.

DDD-lite por módulos: Home, Auth, Profile, Users, Dashboard.
RBAC simples via enum Role (comum, moderador, admin) + cast no User.

Login Google (Socialite) com callback configurado; preserva nome customizado em re-login.
Middleware global RequireAuthentication (redireciona não logados para Home; libera Home e rotas do Google).
Middleware auth custom (redireciona para Home) e role (moderador/admin), com logs.
Logout via POST com CSRF.
Frontend

Inertia + React 18 + TypeScript, Vite 7, Tailwind CSS v4 (@tailwindcss/vite).
Vite com HMR em rede (host 0.0.0.0; HMR 192-168-0-100.nip.io).
Estrutura: resources/js/pages, components/ui, components/layouts.
i18n pt-BR via JSON (home.json, roles.json), importados no TS.
Layout e UI

Layout Inertia (resources/views/app.blade.php) com @inertiaHead e app.tsx.
Header: logo + nome, link para “/”, Dashboard (moderador/admin), avatar como link perfil, botão Google (só ícone no mobile), botão Sair com ícone no desktop, menu hambúrguer no mobile.
Hero: imagem responsiva (banner/bannerWide) com overlay gradiente 60% e CTA.
Cards da Home: busca client-side, grid responsiva, acessibilidade (ARIA, foco, contrastes).
Admin

Dashboard (/admin/dashboard) com sidebar (AdminLayout).
Usuários: lista paginada (20/página), filtros por nome/e-mail/nível, labels amigáveis de nível.
Mobile da lista: tabela com rolagem horizontal e ação “Editar” sempre visível.
Edição: nome e nível; e-mail readonly. Moderador vê admin em somente leitura.
Perfil

Página “/perfil”: edição apenas do nome; e-mail readonly; feedback de sucesso/erro.
Banco e Config

MySQL: DB_HOST=127.0.0.1, DB_DATABASE=projeto-inicial.
Sessão em database; SESSION_DOMAIN=192-168-0-100.nip.io.
Assets públicos em public/img (logo, nome, banner, bannerWide).
Build/Dev

Scripts: npm run dev, vite integrado ao Laravel, bin/serve.sh (artisan serve 0.0.0.0).
TypeScript com paths (@/*) e resolveJsonModule.
Observabilidade e Logs

Monolog padrão (stack); logs adicionais em middlewares (auth/role) e controlador de usuários.
Flash messages exibidas na Home para “Acesso negado” e no Admin para ações.
